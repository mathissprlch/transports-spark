docs/conventions.md §9 taxonomy; one row per fixed bug per §9a.

## Reading the log

- **Tag (a) ratio**: 5 of 7 entries are pure spec misreads. HTTP/2's
  flow control rules (§6.9.1 connection vs §6.9.2 stream, INITIAL_WINDOW_SIZE
  vs WINDOW_UPDATE) are the main trip hazard.
- **(d) pairings** suggest the proof gaps that would have caught them
  at gnatprove time — buffer-disjointness for the WU/`C.Buf` aliasing,
  HPACK table-empty invariant for the decoder leak.
- **Mitigation pattern**: every (a) here was discoverable from RFC 9113
  re-reading. The class would be eliminated by either RecordFlux session
  machines for the flow-control side state, or a hand-mirrored nghttp2
  reference (§5 mitigation for HTTP/2).
